# Публікація та Збереження Will-n-i (How to Publish Forever)

Цей документ описує стратегію публікації, виконання та вічного збереження маніфесту **Will-n-i**.

## 1. Локальне Виконання (Execution)

Щоб зручно читати документацію як веб-сайт локально:

1.  Встановіть залежності (якщо ви хочете гарний інтерфейс):
    ```bash
    npx vitepress dev .
    ```
    Це запустить локальний сервер, де всі `.md` файли стануть гарним сайтом.

2.  Або просто використовуйте **VS Code** з розширенням `Markdown Preview Enhanced`.

## 2. Публікація в Інтернет (Online Publication)

Найпростіший спосіб, який гарантує доступність 24/7 безкоштовно: **GitHub Pages**.

### Кроки:
1.  Ініціалізуйте Git, якщо ще ні (вже зроблено).
2.  Запуште код на GitHub.
3.  У налаштуваннях репозиторію (Settings -> Pages) увімкніть **GitHub Pages** з гілки `main`.
4.  Ваш маніфест буде доступний за адресою `https://your-username.github.io/willni/`.

## 3. Вічне Збереження (Forever in the Internet)

Інтернет забуває, сервери падають, акаунти блокують. Щоб зберегти знання **назавжди**, використовуйте децентралізовані мережі.

### IPFS (InterPlanetary File System)
Зберігання на тисячах комп'ютерів замість одного серверу.

1.  Встановіть [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/).
2.  Імпортуйте папку `willni` в IPFS.
3.  Ви отримаєте **CID** (Content Identifier) — унікальний хеш, наприклад `QmXyZ...`.
4.  Цей хеш неможливо змінити або видалити, доки хоча б один вузол (ви або хтось інший) зберігає ці дані ("pin").
5.  Доступ через шлюзи: `https://ipfs.io/ipfs/QmXyZ...`

### Arweave
Блокчейн для вічного зберігання даних (платите один раз — зберігається назавжди).

1.  Використайте сервіс [ArDrive](https://ardrive.io/) або CLI інструменти.
2.  Завантажте папку.
3.  Отримайте вічне посилання.

## Рекомендований Шлях

1.  **GitHub** — для співпраці та зручного читання зараз.
2.  **IPFS/Arweave** — для історичного архівування версій (наприклад, `v1.0`).
